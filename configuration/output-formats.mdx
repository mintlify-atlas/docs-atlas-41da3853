Metrics can render its output in several formats. The format is controlled by the `config_output` option. The default is `svg`, which produces a scalable vector graphic suitable for embedding in GitHub profile READMEs.

<ParamField path="config_output" type="string" default="auto">
  Output format for the rendered metrics. Accepted values: `svg`, `png`, `jpeg`, `markdown`, `markdown-pdf`, `json`, `insights`.

  When set to `auto`, the format is inferred from the `filename` extension.
</ParamField>

---

## SVG

SVG (Scalable Vector Graphics) is the default and recommended format for GitHub profiles. It preserves text sharpness at any resolution, supports CSS animations, and embeds cleanly in Markdown.

**Use cases:** GitHub profile README, repository README, any Markdown page that renders SVG images.

```yaml SVG output (default)
- uses: lowlighter/metrics@latest
  with:
    token: ${{ secrets.METRICS_TOKEN }}
    filename: github-metrics.svg
    config_output: svg
```

<Tip>
  SVG files load faster than raster images and remain crisp on high-DPI screens. Prefer SVG when possible.
</Tip>

### SVG optimization

<ParamField path="optimize" type="string" default="css, xml">
  Comma-separated list of optimization passes to apply to the SVG output. Accepted values:

  - `css` — minify and purge unused CSS rules
  - `xml` — minify the SVG XML structure
  - `fonts` — subset embedded fonts to only used characters (reduces file size significantly when custom fonts are embedded)

  Set to an empty string to disable all optimization.
</ParamField>

```yaml SVG with all optimizations enabled
- uses: lowlighter/metrics@latest
  with:
    token: ${{ secrets.METRICS_TOKEN }}
    filename: github-metrics.svg
    config_output: svg
    optimize: css, xml, fonts
```

### Self-contained SVGs

<ParamField path="config_base64" type="boolean" default="true">
  Embed all external images (avatars, icons, etc.) as Base64-encoded data URIs. This makes the SVG fully self-contained — it displays correctly without any external network requests.

  Disable when file size is a concern and external image loading is acceptable in your target environment.
</ParamField>

## PNG

PNG renders the SVG to a raster image using a headless browser (Puppeteer). Use PNG when the target platform does not support embedded SVG images.

**Use cases:** Twitter/X cards, email newsletters, platforms that block SVG, embedding in tools that require raster images.

**Requirements:** Puppeteer is included in the metrics Docker image and runs automatically. No additional setup is needed when using the GitHub Action.

```yaml PNG output
- uses: lowlighter/metrics@latest
  with:
    token: ${{ secrets.METRICS_TOKEN }}
    filename: github-metrics.png
    config_output: png
```

<Note>
  PNG files are significantly larger than SVG. Consider using SVG when the platform supports it.
</Note>

---

## JPEG

JPEG applies lossy compression to the rasterized image. Smaller than PNG but with quality loss, especially on text and sharp edges.

**Use cases:** Platforms with strict file size limits where PNG would be too large.

```yaml JPEG output
- uses: lowlighter/metrics@latest
  with:
    token: ${{ secrets.METRICS_TOKEN }}
    filename: github-metrics.jpg
    config_output: jpeg
```

<Warning>
  JPEG lossy compression visibly degrades text and thin lines in metrics output. Use PNG or SVG unless file size is the overriding concern.
</Warning>

## Markdown

The Markdown format outputs a `.md` file instead of an image. It requires the `markdown` template and a Markdown template file.

**Use cases:** Profile READMEs that need dynamic markdown content, documentation pages, wikis.

<ParamField path="markdown" type="string">
  URL or path to the EJS Markdown template file used when `config_output` is `markdown` or `markdown-pdf`. The template is rendered with all metrics data available as variables.

  Supports remote URLs, for example a raw GitHub URL to a template file in another repository.
</ParamField>

<ParamField path="markdown_cache" type="string" default=".cache">
  Directory where embedded SVG renders generated by `embed()` calls inside the Markdown template are cached. These cached files are committed alongside the Markdown output.
</ParamField>

```yaml Markdown output
- uses: lowlighter/metrics@latest
  with:
    token: ${{ secrets.METRICS_TOKEN }}
    template: markdown
    filename: metrics.md
    config_output: markdown
    markdown: >-
      https://raw.githubusercontent.com/your-username/your-repo/main/metrics.template.md
    markdown_cache: .cache
```

<Info>
  The Markdown template engine is [EJS](https://github.com/mde/ejs). Use `{{ variable }}` syntax for escaped output and `<%- %>` for raw HTML. Any data fetched by enabled plugins is available as template variables.
</Info>

## Markdown PDF

Markdown PDF converts the rendered Markdown to a PDF document. This requires the `markdown` template and a Markdown template file.

**Use cases:** Generating a PDF resume or portfolio from GitHub activity data.

```yaml Markdown PDF output
- uses: lowlighter/metrics@latest
  with:
    token: ${{ secrets.METRICS_TOKEN }}
    template: markdown
    filename: metrics.pdf
    config_output: markdown-pdf
    markdown: >-
      https://raw.githubusercontent.com/your-username/your-repo/main/metrics.pdf.template.md
    config_twemoji: true
    config_padding: 5%
```

<Note>
  PDF generation requires Puppeteer, which is included in the metrics Docker image. PDF layout may differ from the Markdown preview depending on CSS print styles in your template.
</Note>

## JSON

The JSON format dumps all collected metrics data as a structured JSON file, without rendering any visual output.

**Use cases:** Debugging available data, building custom tooling on top of metrics data, piping metrics data to other scripts.

```yaml JSON output
- uses: lowlighter/metrics@latest
  with:
    token: ${{ secrets.METRICS_TOKEN }}
    filename: metrics.json
    config_output: json
```

<Tip>
  Use `config_output: json` to discover all available data variables when authoring a Markdown template. The JSON output contains every property exposed to the template engine.
</Tip>

---

## Insights

The Insights format generates an HTML page with interactive charts and statistics about the repository or user. It is intended for use with the metrics web instance rather than GitHub Actions.

**Use cases:** Self-hosted metrics web instance, internal dashboards.

```yaml Insights output (web instance)
- uses: lowlighter/metrics@latest
  with:
    token: ${{ secrets.METRICS_TOKEN }}
    config_output: insights
```

## Display width

<ParamField path="config_display" type="string" default="regular">
  Controls the rendered width for image-based output formats (SVG, PNG, JPEG). Accepted values:

  - `regular` — 480px wide, standard single-column layout
  - `large` — 960px wide, wider layout with more space for content
  - `columns` — multi-column layout that stacks plugins side by side

  This option has no effect on `markdown`, `markdown-pdf`, or `json` outputs.
</ParamField>

```yaml Large display width for PNG output
- uses: lowlighter/metrics@latest
  with:
    token: ${{ secrets.METRICS_TOKEN }}
    filename: github-metrics.png
    config_output: png
    config_display: large
```

## Generating multiple outputs

To produce metrics in multiple formats or with different configurations, define multiple steps in the same workflow job:

```yaml Multiple output formats in one workflow
name: Metrics
on:
  schedule:
    - cron: '0 0 * * *'
  workflow_dispatch:

jobs:
  metrics:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - name: SVG output (GitHub profile)
        uses: lowlighter/metrics@latest
        with:
          token: ${{ secrets.METRICS_TOKEN }}
          filename: github-metrics.svg
          config_output: svg
          config_display: regular

      - name: PNG output (social sharing)
        uses: lowlighter/metrics@latest
        with:
          token: ${{ secrets.METRICS_TOKEN }}
          filename: github-metrics.png
          config_output: png
          config_display: large

      - name: JSON data dump
        uses: lowlighter/metrics@latest
        with:
          token: ${{ secrets.METRICS_TOKEN }}
          filename: metrics-data.json
          config_output: json
```

<Info>
  Each step makes independent API calls. To avoid hitting rate limits when running many steps, use the `delay` option to stagger execution and set appropriate `quota_required_rest` / `quota_required_graphql` thresholds.
</Info>
