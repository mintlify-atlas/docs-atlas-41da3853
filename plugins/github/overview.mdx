GitHub plugins extend your metrics with rich, customizable data pulled directly from the GitHub API. Each plugin focuses on a specific category of information and can be combined freely in a single workflow.

## Enabling plugins

Enable any plugin by adding `plugin_NAME: true` to your workflow step's `with` block:

```yaml
- uses: lowlighter/metrics@latest
  with:
    token: ${{ secrets.METRICS_TOKEN }}
    plugin_languages: yes
    plugin_achievements: yes
    plugin_activity: yes
```

Multiple plugins can be active in the same workflow run. Each enabled plugin appends its section to the generated metrics image.

## Plugin ordering

Use `config_order` to control the sequence in which plugin sections appear in the output:

```yaml
- uses: lowlighter/metrics@latest
  with:
    token: ${{ secrets.METRICS_TOKEN }}
    plugin_languages: yes
    plugin_habits: yes
    plugin_achievements: yes
    config_order: languages, habits, achievements
```

Plugins not listed in `config_order` are appended after the explicitly ordered ones.
