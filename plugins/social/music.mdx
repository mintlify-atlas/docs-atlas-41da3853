---
title: "Music Activity"
description: "Show music listening activity from Spotify, Last.fm, Apple Music, or YouTube Music"
keywords: ["music", "spotify", "lastfm", "apple music", "youtube music", "recently played", "top tracks"]
---

The music plugin displays your listening activity or playlist tracks on your GitHub metrics card. It supports multiple providers and several display modes.

Enable the plugin by adding `plugin_music: true` to your workflow.

## Options

<ParamField path="plugin_music_provider" type="string" required>
  The music service to pull data from. Allowed values: `spotify`, `lastfm`, `apple`, `youtube`.

  This option can be omitted when using `plugin_music_mode: playlist` — the provider will be auto-detected from the playlist URL.
</ParamField>

<ParamField path="plugin_music_token" type="token">
  Authentication credentials for the music provider. The expected format differs by provider:

  - **Spotify**: `"client_id, client_secret, refresh_token"`
  - **Last.fm**: `"api_key"`
  - **YouTube Music**: your browser `cookie` value
  - **Apple Music**: not supported for recent/top modes (playlist mode only)

  Store this as a GitHub secret and reference it with `${{ secrets.YOUR_SECRET_NAME }}`.
</ParamField>

<ParamField path="plugin_music_user" type="string">
  Your username on the music provider. Defaults to your GitHub login. Required for Last.fm to identify your listening history.
</ParamField>

<ParamField path="plugin_music_mode" type="string">
  How to display music data. Defaults to `playlist` if `plugin_music_playlist` is set, otherwise `recent`. Allowed values:

  - `recent` — recently listened tracks
  - `top` — top tracks or artists over a time range
  - `playlist` — random tracks sampled from a playlist URL
</ParamField>

<ParamField path="plugin_music_playlist" type="string">
  An embed playlist URL. Must be an embed URL (the `src` value from an embed iframe, not a regular share link). When set, `plugin_music_mode` defaults to `playlist` and the provider is auto-detected from the URL.
</ParamField>

<ParamField path="plugin_music_limit" type="number">
  Maximum number of tracks to display. Defaults to `4`. Accepted range: 1–100.
</ParamField>

<ParamField path="plugin_music_played_at" type="boolean">
  When enabled, shows the timestamp of when each track was last played. Applies to `recent` mode. Defaults to `false`.
</ParamField>

<ParamField path="plugin_music_time_range" type="string">
  Time range for `top` mode (Spotify only). Defaults to `short`. Allowed values:

  - `short` — approximately 4 weeks
  - `medium` — approximately 6 months
  - `long` — several years
</ParamField>

<ParamField path="plugin_music_top_type" type="string">
  Whether to display top tracks or top artists in `top` mode (Spotify only). Defaults to `tracks`. Allowed values: `tracks`, `artists`.
</ParamField>

## Provider setup

<Tabs>
  <Tab title="Spotify">
    Spotify requires an OAuth flow to obtain a `refresh_token`. This is a one-time setup.

    <Steps>
      <Step title="Create a Spotify app">
        Go to the [Spotify developer dashboard](https://developer.spotify.com/dashboard/) and create a new app. Note your `client_id` and `client_secret`.
      </Step>
      <Step title="Add a redirect URI">
        In your app settings, add `https://localhost` as a redirect URI.
      </Step>
      <Step title="Authorize and get a code">
        Open the following URL in your browser, replacing `YOUR_CLIENT_ID`:

        ```text
        https://accounts.spotify.com/authorize?client_id=YOUR_CLIENT_ID&response_type=code&scope=user-read-recently-played%20user-top-read&redirect_uri=https%3A%2F%2Flocalhost
        ```

        Authorize the app. You will be redirected to `https://localhost?code=AUTHORIZATION_CODE`. Copy the `code` from the URL.
      </Step>
      <Step title="Exchange the code for a refresh token">
        Open the browser console on the Spotify developer dashboard and run:

        ```javascript
        (async () => {
          console.log(await (await fetch("https://accounts.spotify.com/api/token", {
            method: "POST",
            headers: { "Content-Type": "application/x-www-form-urlencoded" },
            body: new URLSearchParams({
              grant_type: "authorization_code",
              redirect_uri: "https://localhost",
              client_id: "YOUR_CLIENT_ID",
              client_secret: "YOUR_CLIENT_SECRET",
              code: "YOUR_AUTHORIZATION_CODE",
            })
          })).json())
        })()
        ```

        Copy the `refresh_token` from the JSON response.
      </Step>
      <Step title="Store credentials as a GitHub secret">
        Add a secret named `SPOTIFY_TOKENS` with the value: `client_id, client_secret, refresh_token`.
      </Step>
    </Steps>

    **Recently played tracks:**

    ```yaml
    name: Spotify recently played
    on:
      schedule:
        - cron: "0 * * * *"
      workflow_dispatch:

    jobs:
      metrics:
        runs-on: ubuntu-latest
        steps:
          - uses: lowlighter/metrics@latest
            with:
              filename: metrics.plugin.music.recent.svg
              token: NOT_NEEDED
              base: ""
              plugin_music: yes
              plugin_music_provider: spotify
              plugin_music_mode: recent
              plugin_music_token: ${{ secrets.SPOTIFY_TOKENS }}
              plugin_music_limit: 4
    ```

    **Top tracks:**

    ```yaml
    name: Spotify top tracks
    on:
      schedule:
        - cron: "0 0 * * *"
      workflow_dispatch:

    jobs:
      metrics:
        runs-on: ubuntu-latest
        steps:
          - uses: lowlighter/metrics@latest
            with:
              token: NOT_NEEDED
              base: ""
              plugin_music: yes
              plugin_music_provider: spotify
              plugin_music_mode: top
              plugin_music_token: ${{ secrets.SPOTIFY_TOKENS }}
              plugin_music_time_range: short
              plugin_music_top_type: tracks
    ```
  </Tab>
  <Tab title="Last.fm">
    Last.fm is the simplest provider to set up — it only requires a free API key.

    <Steps>
      <Step title="Create a Last.fm API account">
        Go to [last.fm/api/account/create](https://www.last.fm/api/account/create) or log in to get an existing API key at [last.fm/api/accounts](https://www.last.fm/api/accounts).
      </Step>
      <Step title="Store the API key as a GitHub secret">
        Add a secret named `LASTFM_TOKEN` with your API key as the value.
      </Step>
    </Steps>

    ```yaml
    name: Last.fm recently played
    on:
      schedule:
        - cron: "0 * * * *"
      workflow_dispatch:

    jobs:
      metrics:
        runs-on: ubuntu-latest
        steps:
          - uses: lowlighter/metrics@latest
            with:
              filename: metrics.plugin.music.recent.svg
              token: NOT_NEEDED
              base: ""
              plugin_music: yes
              plugin_music_provider: lastfm
              plugin_music_token: ${{ secrets.LASTFM_TOKEN }}
              plugin_music_user: your-lastfm-username
              plugin_music_limit: 4
    ```
  </Tab>
  <Tab title="Apple Music">
    Apple Music does not support `recent` or `top` modes due to API access restrictions. Only `playlist` mode is available.

    <Steps>
      <Step title="Get an embed playlist URL">
        On [music.apple.com](https://music.apple.com), open a playlist and select **... → Share → Copy embed code**. Extract the `src` URL from the iframe code.
      </Step>
    </Steps>

    ```yaml
    name: Apple Music playlist
    on:
      schedule:
        - cron: "0 0 * * *"
      workflow_dispatch:

    jobs:
      metrics:
        runs-on: ubuntu-latest
        steps:
          - uses: lowlighter/metrics@latest
            with:
              filename: metrics.plugin.music.playlist.svg
              token: NOT_NEEDED
              base: ""
              plugin_music: yes
              plugin_music_playlist: https://embed.music.apple.com/us/playlist/your-playlist-id
              plugin_music_limit: 4
    ```

    <Note>
      The playlist URL must be the embed `src` URL, not a regular share link.
    </Note>
  </Tab>
  <Tab title="YouTube Music">
    YouTube Music requires extracting a session cookie from your browser.

    <Steps>
      <Step title="Get your session cookie">
        Log in to [music.youtube.com](https://music.youtube.com). Open developer tools (Ctrl+Shift+I), go to the **Network** tab, and filter for `/browse`. Click any matching request, go to the **Headers** tab, and copy the `Cookie` value.
      </Step>
      <Step title="Store the cookie as a GitHub secret">
        Add a secret named `YOUTUBE_MUSIC_TOKENS` with the cookie value.
      </Step>
    </Steps>

    ```yaml
    name: YouTube Music recently played
    on:
      schedule:
        - cron: "0 * * * *"
      workflow_dispatch:

    jobs:
      metrics:
        runs-on: ubuntu-latest
        steps:
          - uses: lowlighter/metrics@latest
            with:
              token: NOT_NEEDED
              base: ""
              plugin_music: yes
              plugin_music_provider: youtube
              plugin_music_mode: recent
              plugin_music_token: ${{ secrets.YOUTUBE_MUSIC_TOKENS }}
    ```
  </Tab>
</Tabs>
