This guide walks you through the minimum steps to generate a metrics SVG and embed it in your GitHub profile README.

<Note>
  You need a GitHub account with a profile repository (`username/username`) to follow this guide. If you don't have one yet, create a new repository with your GitHub username as both the owner and the repository name, and initialize it with a README.
</Note>

<Steps>
  <Step title="Create a GitHub Personal Access Token">
    Metrics needs a token to read your GitHub data via the API.

    1. Go to [github.com/settings/tokens](https://github.com/settings/tokens)
    2. Click **Generate new token (classic)**
    3. Give it a descriptive name, such as `metrics`
    4. Set an expiration date (or select "No expiration" for a permanent token)
    5. Select the following scopes:
       - `read:user` — read user profile data
       - `repo` — read repository data (required for many plugins)
    6. Click **Generate token** and copy the value immediately — you won't be able to see it again

    <Warning>
      Store your token securely. Anyone with this token can read your GitHub data using the selected scopes. Do not commit it to a repository.
    </Warning>
  </Step>

  <Step title="Add the token as a repository secret">
    Your workflow will read the token from a GitHub Actions secret so it is never exposed in code.

    1. Go to your profile repository (`github.com/username/username`)
    2. Click **Settings** → **Secrets and variables** → **Actions**
    3. Click **New repository secret**
    4. Set the name to `METRICS_TOKEN`
    5. Paste the token you generated in the previous step
    6. Click **Add secret**
  </Step>

  <Step title="Create the workflow file">
    Create a new file at `.github/workflows/metrics.yml` in your profile repository with the following contents:

    ```yaml
    name: Metrics
    on:
      schedule:
        - cron: "0 0 * * *"
      workflow_dispatch:
    jobs:
      github-metrics:
        runs-on: ubuntu-latest
        permissions:
          contents: write
        steps:
          - uses: lowlighter/metrics@latest
            with:
              token: ${{ secrets.METRICS_TOKEN }}
    ```

    This workflow runs daily at midnight UTC and can also be triggered manually from the Actions tab. It uses the pre-built Docker image from the GitHub Container Registry by default, so no build step is required.

    <Tip>
      The `workflow_dispatch` trigger lets you run the workflow immediately from the **Actions** tab without waiting for the schedule. This is useful for testing your configuration.
    </Tip>
  </Step>

  <Step title="Push the workflow and let it run">
    Commit and push the workflow file to your profile repository:

    ```bash
    git add .github/workflows/metrics.yml
    git commit -m "Add metrics workflow"
    git push
    ```

    Then go to your repository's **Actions** tab, find the **Metrics** workflow, and click **Run workflow** to trigger it immediately. The workflow generates a file named `github-metrics.svg` and commits it to your repository.

    <Note>
      The first run may take a minute or two as it pulls the Docker image and generates your metrics. Subsequent runs are faster.
    </Note>
  </Step>

  <Step title="Embed the image in your README">
    Once the workflow completes, you'll find `github-metrics.svg` in the root of your profile repository. Add the following line to your `README.md` to display it:

    ```markdown
    ![Metrics](./github-metrics.svg)
    ```

    Push the change to your repository and visit your GitHub profile to see your metrics image.
  </Step>
</Steps>

## Next steps

Now that you have a basic metrics image working, you can customize it further:

- Enable plugins by adding inputs to your workflow — see the [GitHub Action setup guide](/setup/github-action)
- Change the template to alter the overall layout using the `template` input
- Add more data sources by enabling additional plugins with their required tokens
- Self-host a web instance if you want on-demand metrics without GitHub Actions — see the [web instance guide](/setup/web-instance)
